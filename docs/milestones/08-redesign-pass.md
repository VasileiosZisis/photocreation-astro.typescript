# Milestone 08: Redesign Pass

## Goal

Improve the visual design after the Astro migration, SEO, accessibility, and
performance foundations are complete and stable.

## Lesson

Redesign is safer after migration because the content model, routes, metadata,
and static output are already proven. Deployment should come after the redesign
so the first Netlify preview reflects the intended launch design.

## Tasks

### Task 8.1: Review The Current Visual System

Lesson:

Before redesigning, identify what already works.

Check:

- Typography
- Spacing
- Gallery density
- Navigation
- Homepage first impression
- Mobile layout
- Contact conversion path

Expected result:

A short redesign brief with what to keep, change, and remove.

### Task 8.2: Define A New Visual Direction

Lesson:

A photography portfolio should make the images the primary visual asset.

Decide:

- Type scale
- Color palette
- Spacing scale
- Gallery rhythm
- Button/link style
- Motion rules

Expected result:

Small design tokens in CSS custom properties before redesigning components.

### Task 8.3: Redesign One Page First

Lesson:

Test the new direction on one page before changing the whole site.

Recommended page:

Homepage.

Expected behavior:

The homepage feels more polished without breaking content, SEO, or performance.

### Task 8.4: Apply Design System Across Pages

Lesson:

Once one page works, repeat the system. Avoid one-off page styling unless the
content truly needs it.

Expected behavior:

All pages feel related and intentional.

## Done When

- Redesign brief exists
- Design tokens are updated
- Homepage redesign is verified first
- Remaining pages follow the same visual system
- Performance and accessibility are not worse than before
